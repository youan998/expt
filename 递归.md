#### 阶段一：一路“入栈”，寻找第一个序列

因为代码中**入栈的 `if` 块在前面**，所以程序会优先疯狂入栈，直到把 1 到 5 全压进去。

- **Step 1**: current_num = 1 <= 5。执行入栈分支。1 入栈。
    
    - 状态变成：**[2, 栈(底->顶):1, path:空]** -> 递归调用
        
- **Step 2**: current_num = 2 <= 5。执行入栈分支。2 入栈。
    
    - 状态变成：**[3, 栈:1,2, path:空]** -> 递归调用
        
- **Step 3**: current_num = 3 <= 5。3 入栈。
    
    - 状态变成：**[4, 栈:1,2,3, path:空]** -> 递归调用
        
- **Step 4**: current_num = 4 <= 5。4 入栈。
    
    - 状态变成：**[5, 栈:1,2,3,4, path:空]** -> 递归调用
        
- **Step 5**: current_num = 5 <= 5。5 入栈。
    
    - 状态变成：**[6, 栈:1,2,3,4,5, path:空]** -> 递归调用
        
- **Step 6**: current_num = 6，已经 > 5，**跳过入栈分支**。进入出栈分支：栈不为空，弹出栈顶元素 5 放入 path。
    
    - 状态变成：**[6, 栈:1,2,3,4, path:5]** -> 递归调用
        
- **Step 7**: 继续出栈分支（因为 current_num 始终是 6）。弹出 4。
    
    - 状态变成：**[6, 栈:1,2,3, path:5,4]** -> 递归调用
        
- **Step 8-10**: 依此类推，连续出栈 3, 2, 1。
    
    - 状态最终变成：**[6, 栈:空, path:5,4,3,2,1]** -> 递归调用
        
- **Step 11**: 触发递归基准条件 `if (path.size() == 5)`。`totalcount` 变成 1，记录下**第一个合法序列 `[5, 4, 3, 2, 1]`**，并 `return`。
    

---

#### 阶段二：回溯，寻找第二个序列

刚刚找到了第一个序列，程序开始沿着调用栈一层一层往上 `return`，并执行回溯代码（即撤销之前的操作）。

- 当回溯退回到 **Step 5 刚结束入栈分支的时候**（此时 current_num=5，刚尝试完把 5 压栈的所有可能性）：
    
    - **触发回溯**：`s.pop(temp);` 此时的 5 被弹出了，状态恢复到了刚刚准备处理 5 时的样子：**[5, 栈:1,2,3,4, path:空]**。
        
- 接着，程序往下走，遇到了**出栈分支**（`!s.isEmpty()`）：
    
    - 弹出当前栈顶的 4 放入 path。
        
    - 状态变成：**[5, 栈:1,2,3, path:4]** -> 递归调用
        
- 现在进入了新的递归（在 path 已经有 4 的情况下）：
    
    - 此时 current_num = 5 <= 5。执行入栈分支。5 入栈。
        
    - 状态变成：**[6, 栈:1,2,3,5, path:4]** -> 递归调用
        
- 接下来 current_num 变成了 6，只能疯狂出栈：
    
    - 弹出 5 -> path 变成 `[4, 5]`
        
    - 弹出 3 -> path 变成 `[4, 5, 3]`
        
    - 弹出 2 -> path 变成 `[4, 5, 3, 2]`
        
    - 弹出 1 -> path 变成 `[4, 5, 3, 2, 1]`
        
- 满足 `path.size() == 5`，`totalcount` 变成 2，记录下**第二个合法序列 `[4, 5, 3, 2, 1]`**，并 `return`。

#### 阶段三：穷举所有的可能性

程序会像这样，不知疲倦地“压栈 -> 尝试 -> 到底 -> 记录 -> 撤销（回溯） -> 尝试另一种操作”，最终遍历所有的分支。
